In Windows, `\includegraphics{directory/file}` leads to the xml `<graphics candidates="directory\file.png" graphic="directory/file"/>`,
(possibly from lib/LaTeXML/Package/graphicx.sty.ltxml)
which leads to the html `<img src="directory%5Cfile.png">`.
(possibly from lib/LaTeXML/resources/XSLT/LaTeXML-misc-xhtml.xsl).
%5C is the url encoding for \, but the html needs / if it is going to find the image.  It turns out that a literal \ also works for local html files, but I doubt that would make for a portable website when I copy everything over to a server.
